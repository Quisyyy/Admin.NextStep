üìù Tables and Functions

The audit trail setup creates:

  • audit_trail table - Stores all action logs with user info, action type, entity details
  • log_audit_trail function - Helper function to insert audit records
  • Updated RLS policies - Ensures admin users can view audit logs
  • Indexes - For fast querying of audit records

üìä What Gets Logged

Each audit record includes: - User ID and email - Action type (ARCHIVE, RESTORE, DELETE, UPDATE, CREATE, LOGIN) - Entity type (ALUMNI, FORM, CAREER, ADMIN) - Entity ID and name - Description of the action - Old and new values (for updates) - Timestamp with timezone - Additional metadata - User agent information
